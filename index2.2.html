00 — CONTEXTO

¿Por qué existen los modos?

El cifrado simétrico usa una sola clave para cifrar y descifrar. AES procesa datos en bloques fijos de 128 bits. El problema: el mundo real tiene archivos enormes, transmisiones continuas y estructuras complejas.

Los modos de operación son el puente que define cómo aplicar AES de forma segura a cualquier cantidad de datos.

01 — MODO DE OPERACIÓN

INSEGURO

ECB

Electronic Codebook

Cada bloque se cifra de forma independiente con la misma clave. El modo más simple — y el más peligroso.

El problema fatal: bloques de texto plano idénticos producen bloques cifrados idénticos. Estructuras y patrones permanecen visibles — el famoso efecto pingüino.

✓ Rápido — procesamiento paralelo
✗ No oculta patrones — vulnerable a análisis
✗ Propenso a ataques de repetición

TEXTO PLANO →

HOLA HOLA HOLA HOLA

TEXTO CIFRADO ECB →

A3F2 A3F2 A3F2 A3F2

— patrones repetidos, completamente visibles

02 — MODO DE OPERACIÓN

ESTÁNDAR

CBC

Cipher Block Chaining

Antes de cifrar, cada bloque recibe un XOR con el bloque cifrado anterior. Una cadena de dependencias que elimina los patrones.

El primer bloque usa un Vector de Inicialización (IV) único. Textos idénticos producen cifrados diferentes. Un error en un bloque afecta los siguientes.

✓ Oculta patrones completamente
✓ El IV garantiza unicidad del cifrado
✗ No permite cifrado paralelo
✗ Propagación de errores en cadena

03 — MODO DE OPERACIÓN

FLUJO

CFB

Cipher Feedback

Transforma el cifrado de bloque en un cifrado de flujo. El texto cifrado anterior alimenta la siguiente operación de cifrado.

Ideal para transmitir datos cuyo tamaño no sea múltiplo del bloque (128 bits). Maneja flujos de cualquier longitud con seguridad.

✓ Datos de tamaño arbitrario
✓ Útil en transmisiones en tiempo real
✗ Propagación de errores (como CBC)

04 — MODO DE OPERACIÓN

ROBUSTO

OFB

Output Feedback

El feedback proviene de la salida directa del cifrador, no del texto cifrado. Genera un flujo de claves completamente independiente.

Gran ventaja: errores en la transmisión de un bit no se propagan a bloques siguientes. Usado en comunicaciones por satélite donde los errores son inevitables.

✓ Sin propagación de errores de bit
✓ Flujo de claves pre-generado
⚠ Requiere IV único (nunca reutilizar)

05 — MODO DE OPERACIÓN

MODERNO ★

CTR

Counter Mode

Un contador + nonce se cifra y el resultado se aplica con XOR al texto plano. No depende de bloques anteriores.

El modo más moderno y eficiente. Paralelismo total: cifra y descifra múltiples bloques simultáneamente en procesadores multinúcleo. Estándar para discos, VPNs y redes de alta velocidad.

✓ Paralelismo total (Multi-core)
✓ Rápido y seguro
✓ Sin propagación de errores
✓ Acceso aleatorio a cualquier bloque

CASOS REALES

En producción hoy

🌐

HTTPS / TLS

Navegación web segura. TLS 1.3 usa AES-GCM (basado en CTR) para cifrar cada sesión.

🔒

VPN

WireGuard y OpenVPN usan CTR y CBC para proteger datos en tránsito sobre internet.

💾

CIFRADO DE DISCO

BitLocker y LUKS usan variantes de CTR (XTS-AES) para cifrar discos completos con acceso aleatorio.

📡

SATÉLITE / RADIO

OFB se usa en canales ruidosos donde los errores de bit son frecuentes y la propagación es inaceptable.

CONCLUSIÓN

Un algoritmo,
infinitas formas
de usarlo.

Los modos de operación son el puente entre la teoría del cifrado de bloques y la realidad técnica. Cada modo resuelve un problema distinto: ECB muestra por qué el contexto importa, CBC introduce la dependencia, CFB/OFB adaptan bloques a flujos, y CTR lleva AES al hardware moderno.

ECB — Simple CBC — Encadenado CFB — Flujo OFB — Robusto CTR — Moderno ★
